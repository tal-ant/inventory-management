Architecture overview

Factory Inventory Management System

A workshop demo for Catalyst Components: inventory, orders, demand forecasting, backlog, and spending analytics for factory operations. A Vue 3 single-page app talks to a FastAPI service that filters in-memory mock data loaded from JSON files — there is no database and no authentication.

Data flow — one filtered request

Browser
01
FilterBar.vue
The user picks Time Period, Location, Category, or Order Status. Filters apply globally to every view.
02
composables/useFilters.js
Selections land in shared module-level refs (a singleton), so every view sees the same filter state.
03
views/*.vue
Each view watches the filters and reloads its data, passing getCurrentFilters() to the API client.
04
api.js
Axios issues GET /api/orders?warehouse=…&category=…&status=…&month=…, omitting any filter set to "all".
Server
05
main.py — endpoint
FastAPI parses the query params and runs apply_filters() and filter_by_month() over the in-memory lists (plain list comprehensions).
06
Pydantic response_model
The filtered dicts are validated and serialized against the response model, then returned as JSON.
Browser
07
views/*.vue — refs
Raw arrays go into refs (allOrders, inventoryItems); KPIs and chart series are derived in computed properties.
08
Templates
Custom SVG charts and CSS-grid tables re-render reactively. Currency and labels go through useI18n (English/Japanese, USD/JPY).

Things to know before changing it

api.js defines methods for /api/tasks and /api/purchase-orders, but main.py has no matching routes — those calls would 404. The Tasks modal currently reads mock tasks from useAuth instead, and CreatePurchaseOrderRequest is defined but unused.
Backlog.vue exists under views/ but is not registered in the router; backlog data surfaces through the dashboard and its detail modal.
Inventory has no time dimension, so the month/quarter filter applies only to orders and order-derived numbers, not inventory queries.
When a JSON data file's structure changes, the matching Pydantic model in main.py must change too, or responses fail validation.
Demo posture by design: in-memory data (resets on restart), CORS open to all origins, no authentication, no pagination. Revenue goals are fixed at $800K/month and $9.6M YTD.
